\doxysubsection{Detailed Description}
Provides rect/rubber-\/band data selection and range zoom interaction. 

\doxylink{class_q_c_p_selection_rect}{QCPSelection\+Rect} is used by \doxylink{class_q_custom_plot}{QCustom\+Plot} when the \doxylink{class_q_custom_plot_a810ef958ebe84db661c7288b526c0deb}{QCustom\+Plot\+::set\+Selection\+Rect\+Mode} is not \doxylink{namespace_q_c_p_ac9aa4d6d81ac76b094f9af9ad2d3aacfa9032f170490d67240a6c68c2638ffab1}{QCP\+::srm\+None}. When the user drags the mouse across the plot, the current selection rect instance (\doxylink{class_q_custom_plot_a0c09f96df15faa4799ad7051bb16cf33}{QCustom\+Plot\+::set\+Selection\+Rect}) is forwarded these events and makes sure an according rect shape is drawn. At the begin, during, and after completion of the interaction, it emits the corresponding signals \doxylink{class_q_c_p_selection_rect_a7b7162d19f4f2174d3644ff1a5d335aa}{started}, \doxylink{class_q_c_p_selection_rect_a1bab11026bca52740c2e6682623e6964}{changed}, \doxylink{class_q_c_p_selection_rect_aeb82009393c90130102dccf36477b906}{canceled}, and \doxylink{class_q_c_p_selection_rect_a15a43542e1f7b953a44c260b419e6d2c}{accepted}.

The \doxylink{class_q_custom_plot}{QCustom\+Plot} instance connects own slots to the current selection rect instance, in order to react to an accepted selection rect interaction accordingly.

\doxylink{class_q_c_p_selection_rect_ad27c1569c6ea8fa48e24b81e2a302df3}{is\+Active} can be used to check whether the selection rect is currently active. An ongoing selection interaction can be cancelled programmatically via calling \doxylink{class_q_c_p_selection_rect_af67bc58f4f5ce9a4dc420b9c42de235a}{cancel} at any time.

The appearance of the selection rect can be controlled via \doxylink{class_q_c_p_selection_rect_ada20b7fb1b2dcbe50523262636b06963}{set\+Pen} and \doxylink{class_q_c_p_selection_rect_ab0c66f1484418782efa01f4153611080}{set\+Brush}.

If you wish to provide custom behaviour, e.\+g. a different visual representation of the selection rect (\doxylink{class_q_c_p_selection_rect_ab0e50ae796508bdcd97ab8c335c593bf}{QCPSelection\+Rect\+::draw}), you can subclass \doxylink{class_q_c_p_selection_rect}{QCPSelection\+Rect} and pass an instance of your subclass to \doxylink{class_q_custom_plot_a0c09f96df15faa4799ad7051bb16cf33}{QCustom\+Plot\+::set\+Selection\+Rect}.

{\footnotesize\ttfamily void QCPSelection\+Rect\+::apply\+Default\+Antialiasing\+Hint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_q_c_p_painter}{QCPPainter}} \texorpdfstring{$\ast$}{*}}]{painter }\end{DoxyParamCaption}) const\hspace{0.3cm}{\ttfamily [protected]}, {\ttfamily [virtual]}}

This function applies the default antialiasing setting to the specified {\itshape painter}, using the function \doxylink{class_q_c_p_layerable_acb663e375d2d36dc5c55021ee5a2119b}{apply\+Antialiasing\+Hint}. It is the antialiasing state the painter is put in, when \doxylink{class_q_c_p_selection_rect_ab0e50ae796508bdcd97ab8c335c593bf}{draw} is called on the layerable. If the layerable has multiple entities whose antialiasing setting may be specified individually, this function should set the antialiasing state of the most prominent entity. In this case however, the \doxylink{class_q_c_p_selection_rect_ab0e50ae796508bdcd97ab8c335c593bf}{draw} function usually calls the specialized versions of this function before drawing each entity, effectively overriding the setting of the default antialiasing hint.

{\bfseries{First example\+:}} \doxylink{class_q_c_p_graph}{QCPGraph} has multiple entities that have an antialiasing setting\+: The graph line, fills and scatters. Those can be configured via \doxylink{class_q_c_p_layerable_a4fd43e89be4a553ead41652565ff0581}{QCPGraph\+::set\+Antialiased}, \doxylink{class_q_c_p_abstract_plottable_a089d6b5577120239b55c39ed27c39536}{QCPGraph\+::set\+Antialiased\+Fill} and \doxylink{class_q_c_p_abstract_plottable_a2f03f067ede2ed4da6f7d0e4777a3f02}{QCPGraph\+::set\+Antialiased\+Scatters}. Consequently, there isn\textquotesingle{}t only the \doxylink{class_q_c_p_layerable_afdf83ddc6a265cbf4c89fe99d3d93473}{QCPGraph\+::apply\+Default\+Antialiasing\+Hint} function (which corresponds to the graph line\textquotesingle{}s antialiasing), but specialized ones like \doxylink{class_q_c_p_abstract_plottable_a8d06a59ea23324cce6330ebf2262c0ed}{QCPGraph\+::apply\+Fill\+Antialiasing\+Hint} and \doxylink{class_q_c_p_abstract_plottable_ac95f26b15a1e5d9c7bd2c0a46d760fc9}{QCPGraph\+::apply\+Scatters\+Antialiasing\+Hint}. So before drawing one of those entities, \doxylink{class_q_c_p_graph_a2b0849598f06e834b43ce18cd13bcdc3}{QCPGraph\+::draw} calls the respective specialized apply\+Antialiasing\+Hint function.

{\bfseries{Second example\+:}} \doxylink{class_q_c_p_item_line}{QCPItem\+Line} consists only of a line so there is only one antialiasing setting which can be controlled with \doxylink{class_q_c_p_layerable_a4fd43e89be4a553ead41652565ff0581}{QCPItem\+Line\+::set\+Antialiased}. (This function is inherited by all layerables. The specialized functions, as seen on \doxylink{class_q_c_p_graph}{QCPGraph}, must be added explicitly to the respective layerable subclass.) Consequently it only has the normal \doxylink{class_q_c_p_layerable_afdf83ddc6a265cbf4c89fe99d3d93473}{QCPItem\+Line\+::apply\+Default\+Antialiasing\+Hint}. The \doxylink{class_q_c_p_item_line_ae184140b61b2ef5b8edde76304447200}{QCPItem\+Line\+::draw} function doesn\textquotesingle{}t need to care about setting any antialiasing states, because the default antialiasing hint is already set on the painter when the \doxylink{class_q_c_p_selection_rect_ab0e50ae796508bdcd97ab8c335c593bf}{draw} function is called, and that\textquotesingle{}s the state it wants to draw the line with.
